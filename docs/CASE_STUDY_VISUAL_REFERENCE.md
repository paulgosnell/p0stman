## TYPOGRAPHY SCALE & SPACING

### Font Hierarchy

```
Title (Hero):         4xl-7xl    (~48px-84px)   Playfair Display, light
Section Title:        5xl-7xl    (~48px-84px)   Playfair Display, light
Heading (H2):         3xl-4xl    (~30px-36px)   Playfair Display, light
Stat Metric:          3xl-4xl    (~60px-72px)   Playfair Display, light
Body Large:           xl-2xl     (~20px-24px)   Light weight
Body Regular:         lg-xl      (~18px-20px)   Light weight
Small Text:           sm-base    (~12px-16px)   Light weight
Pill/Badge:           xs         (~12px)        Light weight
```

### Line Height Impact on Text Length

```
Line Height 1.1 (tight):
"Traditional Health Information Crisis" → Wraps efficiently
Perfect for: Titles, headlines (30-50 chars ideal)

Line Height 1.5 (normal):
"People are dying from confusion, not disease..." → Scannable
Perfect for: Body text, descriptions (80-200 chars ideal)

Line Height 2 (relaxed):
"2-3 week build vs. 6-month traditional..." → Maximum readability
Perfect for: Large body text, quotations (150-400 chars ideal)
```

## COMMON PITFALLS & SOLUTIONS

### Problem 1: Titles Too Long
```
✗ "Building an AI-Powered Enterprise Digital Transformation Platform"
  (65 chars - wraps excessively, loses impact)

✓ "Enterprise AI Transformation"
  (28 chars - crisp, clear, memorable)
```

### Problem 2: Descriptions Too Short
```
✗ "Transforming healthcare through AI."
  (36 chars - insufficient detail, seems rushed)

✓ "Mamori built an AI-native operating system that aggregates fragmented 
  health data, applies evidence-based analysis, and delivers personalized 
  guidance through natural conversational interfaces."
  (265 chars - compelling narrative)
```

### Problem 3: Bullet Points Too Long
```
✗ "The mobile experience was significantly impacted by slow loading times 
  and poor user interface responsiveness on lower bandwidth connections"
  (144 chars - too detailed for bullet)

✓ "Mobile loading times exceeded 8 seconds, impacting user engagement"
  (65 chars - concise, specific, scannable)
```

### Problem 4: Stats Labels Too Vague
```
✗ "Performance"      (11 chars - unclear what metric)
✗ "Team"             (4 chars - ambiguous)

✓ "Response Time"    (13 chars - specific, clear)
✓ "Team Growth"      (10 chars - contextual)
```

### Problem 5: Technologies List Too Long
```
✗ Lists with 20+ items create vertical scroll fatigue

✓ Curate to 6-12 most relevant/impressive technologies
  Prioritize: Frontend, Backend, AI, Infrastructure
```

## EXAMPLE CONTENT VARIATIONS

### For Small Projects (Low Complexity)
```
Hero Title:           "Mobile App MVP"              (14 chars)
Description:          80-100 chars (shorter)
Challenge Points:     3 points (simpler)
Benefits Stats:       3-4 items (focused)
Solution Approach:    2-3 steps (streamlined)
```

### For Enterprise Projects (High Complexity)
```
Hero Title:           "Enterprise Platform Suite"   (25 chars)
Description:          150-200 chars (detailed)
Challenge Points:     4-5 points (comprehensive)
Benefits Stats:       6 items (detailed metrics)
Solution Approach:    4-5 steps (thorough)
```

### For AI/ML Projects
```
Emphasize in Solution Description:
- ML models/algorithms
- Training data scale
- Inference speed
- Accuracy metrics

Example: "Built Claude 3.5 Sonnet Vision model integration with custom 
vector embeddings for semantic search across 10M+ health records with 
<500ms query latency and 94% accuracy."
```

## COPYWRITING FORMULA TEMPLATES

### Challenge Section
```
[Problem Headline]: [Specific Issue]
↓
[Opening Sentence]: [Root Cause or Impact Statement]
↓
• [Consequence 1 with Statistic]
• [Consequence 2 with Statistic]
• [Consequence 3 with Market Reality]
• [Consequence 4 with User Impact]

Example:
"Traditional Health Information Crisis"
↓
"People are dying from confusion, not disease."
↓
• "Information Overload Paralysis: 1 in 4 consumers overwhelmed"
• "Trust Deficit: 73% distrust health information"
• "Fragmented Data: Multiple apps that don't communicate"
• "Low Health Literacy: 25% higher mortality rates"
```

### Solution Section
```
[Tech Stack]: [Specific Technologies]
↓
[Architecture]: [How it Works - Narrative]
↓
01 [First Implementation Step]
02 [Second Implementation Step]
03 [Third Implementation Step]
04 [Fourth Implementation Step]
↓
"[Key Achievement or Outcome]"

Example:
"React 19, Supabase, Claude Vision, 11Labs"
↓
"Built an AI-native operating system that aggregates fragmented 
health data, applies evidence-based analysis, and delivers 
personalized guidance through natural conversational interfaces."
↓
01 "Aggregated data from wearables, lab results, and behavior tracking"
02 "Applied proprietary AI Evidence Engine analyzing clinical trials"
03 "Delivered personalized guidance through conversational interfaces"
04 "Scaled to handle millions of concurrent health profile queries"
↓
"Created unified health trust layer eliminating information paralysis"
```

### Benefits Section
```
[Metric Number]: [Impact Description]
↓
[Supporting Context or Comparison]

Example:
"95% cost reduction"
↓
"Built production-ready MVP for <€5K vs. traditional €300-500K 
agency spend through AI-augmented solo development"

---

"300% faster time-to-market"
↓
"2-3 week build vs. 6-month traditional development cycle, enabling 
rapid user feedback loops and product-market fit iteration"

---

"Sub-2s AI response times"
↓
"Claude Vision analysis and voice AI responses deliver insights faster 
than traditional human consultation wait times"
```
